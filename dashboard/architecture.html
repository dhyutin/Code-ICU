How each platform is used

Hugging Face

  • Source of the real training data via datasets.load_dataset
  • rotten_tomatoes (text) and mnist (vision) example tasks
  • Working + intentionally-buggy *_fail.py variants for the demo

PyTorch

  • The actual model training (small MLPs, real autograd)
  • Produces the genuine loss / gradient-norm signal we monitor
  • The bugs (high LR, missing normalization) cause real divergence

InsForge

  • Postgres backend: run_logs, runs, profiles, agents, call_decisions, agent_events, fix_attempts
  • Google OAuth auth + per-user Row-Level Security
  • Agent registry (prompts/models live in the DB, not code)
  • SDK powers the dashboard reads and the Restore action

Nebius

  • LLM gateway (OpenAI-compatible) for all five agents
  • Code understanding, error classification, call gating
  • Distills the call transcript into a single decision
  • Proposes the minimal code patch in the self-healing loop

Vapi

  • Places the outbound phone call to the researcher
  • Hosts agent 5 as a live voice assistant
  • Receives run context via variableValues and returns the transcript

code_icu core

  • Heuristic anomaly detection + agent orchestration
  • Backups, changes.txt, rollback watcher
  • Glue between PyTorch, InsForge, Nebius, and Vapi
